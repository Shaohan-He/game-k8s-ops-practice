## 四、问题一：8080 端口被 Kuboard 占用

### 1. 排查命令

```bash
ss -lntp | grep ':8080'
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}" | grep -E "8080|8000|8001|8002|8003|kuboard|game|gateway|login|match|room|nginx"
```

### 2. 排查结果

发现 `8080` 端口被 Kuboard 容器占用：

```text
kuboard    Up    0.0.0.0:8080->80/tcp
```

继续访问：

```bash
curl -i http://localhost:8080/login
```

返回结果中出现：

```text
HTTP/1.1 303 See Other
Location: /sso/auth?access_type=offline&client_id=kuboard-sso...
```

### 3. 原因分析

项目默认使用 `8080` 作为 Nginx 业务入口，但当前 `master01` 节点上已经运行了 Kuboard，并占用了宿主机 `8080` 端口。

![](端口占用.png)

因此访问：

```text
http://localhost:8080
```

实际进入的是 Kuboard，而不是本项目的 Nginx。

### 4. 解决方式

将 `docker-compose.yml` 中 Nginx 的端口映射从：

```yaml
8080:80
```

修改为：

```yaml
18080:80
```

执行重启：

```bash
docker compose down
docker compose up -d
```

### 5. 处理结果

后续所有业务接口统一改为：

```text
http://localhost:18080
```

不再使用：

```text
http://localhost:8080
```

## 五、问题二：game-gateway 启动失败

### 1. 启动现象

执行：

```bash
docker compose up -d
```

出现：

```text
dependency game-gateway failed to start
container game-k8s-ops-practice-game-gateway-1 is unhealthy
```

查看容器状态时发现：

```text
game-gateway    Restarting / unhealthy
```

### 2. 排查命令

```bash
docker logs --tail=200 game-k8s-ops-practice-game-gateway-1
```

### 3. 关键报错

日志中出现：

```text
RuntimeError: 'cryptography' package is required for sha256_password or caching_sha2_password auth methods
```

完整报错链路显示：

```text
game-gateway 启动
  ↓
FastAPI lifespan 阶段初始化
  ↓
infrastructure.connect()
  ↓
aiomysql.create_pool()
  ↓
连接 MySQL 8.4
  ↓
MySQL 使用 caching_sha2_password 认证方式
  ↓
Python 镜像缺少 cryptography 包
  ↓
应用启动失败
```

### 4. 原因分析

项目使用 MySQL 8.4，默认认证方式为 `caching_sha2_password`。

业务服务使用 `aiomysql / PyMySQL` 连接 MySQL 时，如果使用该认证方式，需要 Python 环境中安装 `cryptography` 依赖。

当前业务镜像中的依赖文件没有包含 `cryptography`，导致 `game-gateway` 在启动阶段连接 MySQL 失败，最终容器反复重启。

### 5. 解决方式

在 `requirements.txt` 中补充依赖：

```text
cryptography>=42.0.0
```

然后重新构建业务镜像：

```bash
IMAGE_REGISTRY=game-k8s-ops-practice IMAGE_TAG=1.0.0 bash scripts/build-images.sh
```

或者使用 Compose 重新构建：

```bash
docker compose build --no-cache game-gateway login-service match-service room-service
```

重新启动：

```bash
docker compose up -d --force-recreate game-gateway login-service match-service room-service
docker compose up -d nginx
```

### 6. 处理结果

重新构建并启动后，`game-gateway`、`login-service`、`match-service`、`room-service` 均恢复正常。

- 四个 FastAPI 服务均已通过健康检查。
